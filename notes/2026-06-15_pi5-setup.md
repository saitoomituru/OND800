## Season 1ゴール確認(README記載と一致)
最小単位: 電源投入→USBカメラをudev/v4l2で自動検出→各カメラ最大解像度/fpsでNDIストリーム化

### 検証すべき技術的キーポイント
- GStreamerの`ndisink`プラグインがPi5上でビルド/動作するか(NDI SDK ARM64対応状況含む)
- 動作確認できれば、Pythonがプロセスマネージャとして各カメラごとにGStreamerパイプラインを
  subprocessで起動・監視する構成へ進む(1カメラ死んでも他に影響しないプロセス分離)

### 構成イメージ
```
v4l2-ctl / pyudev    → カメラ検出・hotplug
GStreamer + ndisink  → カメラ映像→NDI変換(プロセスとして起動)
LVGL (C)             → 操作パネル(HyperPixel 4.0, 別プロセス)
libwebsockets        → obs-websocket連携
Python               → オーケストレーション層(検出→プロセス起動・管理、UIとはローカルHTTP/WSで疎結合)
```

## ハードウェア注記(README補足)
- Pi5 CPU(BCM2712)/RP1/PMIC(ルネサス)へのヒートシンク対応済み
- バッテリーHAT検討時はPi5対応モデル(PiSugar 3系等、USB-C PD入力)を選定すること。
  Pi4世代のUPS HATは電力不足の懸念あり
